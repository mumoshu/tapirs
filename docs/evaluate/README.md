**P1 — Purpose:** You're evaluating whether tapirs is the right distributed transactional store for your project. This section gives you the information you need to make that decision — what tapirs offers and how it compares to alternatives you may already be considering.

**P2 — Reading order:** Start with [Benefits](benefits.md) to understand what makes tapirs different: leaderless transactions with no single point of failure, self-contained deployment with no external dependencies, and online resharding that grows with your workload. Then check [Use Cases](usecases.md) to see if tapirs fits your specific scenario — from SQL database backends to stateful microservices — and read the [Comparison](comparison.md) to see how it stacks up against etcd, FoundationDB, CockroachDB, and TiKV.

**P3 — Next steps:** If you've already decided to try tapirs, jump straight to [Getting Started](../operate/getting-started-testbed.md) — you can have a cluster running in under a minute. Back to [Overview](../README.md).
